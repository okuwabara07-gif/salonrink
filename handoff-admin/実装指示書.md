# SalonRink 管理画面 ニューモーフィズム再デザイン — 実装指示書（Claude Code 用）

これは salonrink.com の**ログイン後 管理画面**を、同梱モックのデザインに合わせて再実装するための指示書です。
同梱の `SalonRink 管理画面 Neo.html` + `admin-neo.css` が**確定デザインの実物リファレンス**です。迷ったら必ずこのモックの見た目・数値に合わせてください。

## 1. デザインシステム（厳守）

### トークン（admin-neo.css 冒頭の `.ad {}` がソース・オブ・トゥルース）
- ベース: `--bg`（ベージュ #EFE8E2 / ホワイト #F6F5F2 の2種）
- 影: `--dark` / `--light` のペア。**浮き出し** `--raise`(7px/14px) と **凹み** `--inset`(4px/8px) の2段
- アクセント: `--acc` / `--acc-ink` / `--grad` — terracotta / sage / bluegray / gold / plum / charcoal の6種
- フォント: Zen Kaku Gothic New（本文400-700・見出し900）+ Caveat（手書き英字アクセント・数字）
- `--bg-main` / `--dark-main` はテーマ原値のエイリアス（レイヤー内で `--bg` を再定義しても参照できるように）

### ニューモーフィズムの原則
- 面は全て `--bg` と同色 + 影で凹凸を作る（ベタ塗りの面・枠線は使わない）
- ボタン/ピル = 浮き出し、入力欄/トラック/タブの選択面 = 凹み
- アクティブ状態 = `--grad`（グラデ塗り+白文字）
- 白抜き文字はアクティブ要素のみ

### レイアウト（スプリット構成・確定）
- 背景はプレーンな `--bg`。その上に**2枚の浮きカード**:
  - サイドバー: 幅232px・角丸26px・sticky（`top: 20px; height: calc(100vh - 40px)`）
  - メイン: 角丸26px・`min-height: calc(100vh - 40px)`
- `.ad { padding: 20px; gap: 26px }`、メイン左 `-14px` に**縦の仕切り線**（1.5px・`color-mix(ink 18%)`・カードの上下より20px長い）

## 2. テーマ切替（実装必須）
- `data-theme`（ベース: 無印=ベージュ / `white`）× `data-acc`（6色）の2軸。CSS変数の差し替えのみで全UIが追従する
- 選択は永続化（モックは localStorage キー `sr-admin-base` / `sr-admin-acc`。本実装ではユーザー設定として保存先は任意）
- 既定: ベージュ × terracotta

## 3. 画面構成（10ビュー）
モックのサイドバー順: ホーム / HPB予約 / 顧客 / DM配信 / 連携 / コンシェルジュ(NEW) / プラン(FREE) / お知らせ(3) / 店販EC / 口コミ承認

| ビュー | 要点 |
|---|---|
| ホーム | KPI4枚（数値34px・Caveatではなく Zen Kaku 900）+ 本日のタイムライン + お知らせ3件 |
| HPB予約 | 日付ナビ（Caveat大字）+ 凹みレーンのタイムテーブル + 予約ブロック（グラデ=指名 / 浮き出し=フリー枠） |
| 顧客 | 絞り込みチップ + 一覧（PC=5列グリッド表 / SP=カード型で**名前26px**） |
| DM配信 | テンプレ6種（選択=凹み+アクセント縁）→ セグメントチップ → プレビュー(凹み面+白バブル) → 配信サマリー+CTA |
| 連携 | 2列カード（LINE=接続中 / HPB / リッチメニュー / Googleカレンダー） |
| コンシェルジュ | KPI3枚のサマリー |
| プラン | トライアル(現在)/Light ¥1,980/Standard ¥2,980 の3枚。現在=凹み+縁 |
| お知らせ | カテゴリチップ + 記事カード |
| 店販EC | タブ3つ: 注文管理 / 商品一覧 / **商品登録**（下記） |
| 口コミ承認 | 空状態（凹み面） |

### 店販EC 商品登録フォーム
商品名 / カテゴリ(select) / 価格(円・税込) / 在庫(個) / 品番 / バリエーション（チップ追加・×削除）/ 商品説明(textarea) / 画像（正方形タイル・最大5枚・D&D）/ トグル3つ（ストアに公開・カウンセリングで提案・在庫連動）。
入力欄は全て凹み(`--inset-sm`)、登録CTAは `--grad` のピル。

## 4. レスポンシブ（760px 境界）
- PC: スプリット2カード+サイドバー
- SP: カード化解除（フラット・角丸0）、サイドバー非表示 →
  - **下部ピルナビ**（ホーム/予約/顧客/DM配信/その他、`env(safe-area-inset-bottom)` 対応）
  - 「その他」= ボトムシートで残り6ビュー
  - 検索バー全幅 / KPI数値30px / 顧客一覧はカード型・名前26px
- ナビのアクティブ状態はサイドバー・下部ナビ・シートの3箇所で同期（モックの `go()` 関数参照）

## 5. 日本語の折り返し対策（重要・モックで解決済み）
日本語ラベルは `white-space: nowrap` がないとピル内で折れて崩れる。モックでは以下に nowrap(+ellipsis) 適用済み — 流用すること:
`.chip` `.tl-row .m/.st` `.cust-id strong/span` `.cust-num` `.dm-sum b` `.int-st` `.int-go` `.v-head h2/.right` `.day-nav .big/.d` `.topbar .date` `.search` `.tt-staff strong/span` `.int-head span` `.prod-row strong/.m/.st`

## 6. 実装上の注意
- `color-mix(in srgb, ...)` を多用している。対象ブラウザが古い場合はビルド時に静的展開するか PostCSS で代替
- 既存の管理画面はダークブラウンのサイドバー。**全置き換え**であり部分適用しない（混在すると世界観が崩れる）
- データ結線は既存APIのまま。モック内の数値・顧客名・注文はすべてダミー
- エラー時は赤帯ではなく、モックの `.empty`（凹み面の空状態）+ 再試行ボタンのトーンで
- 文字サイズの最小: 補足10px / 本文12px台 / 見出し17-20px / KPI 34px
