## 🚨 営業開始における課題

### 緊急対応が必要な項目

#### 1. **LINE リマインド機能（高優先度）**
- **問題**: LP で大訴求「リマインド送信」の実装が不完全
- **現状**: API スケルトンのみ、送信ロジックなし
- **影響**: 営業時に「自動リマインド」を訴求しているが、実装されていない
- **対応**: 
  - [ ] `app/api/cron/reminder/route.ts` 実装 (Cloud Scheduler 統合)
  - [ ] 前日 18:00 に LINE メッセージ自動送信
  - [ ] Supabase Edge Functions または外部スケジューラ設定

#### 2. **ホットペッパー同期（高優先度）**
- **問題**: UI 実装のみ、実際の HPB API 連携なし
- **現状**: 認証情報は保存されるが、予約データが同期されない
- **影響**: 営業開始後「ホットペッパー予約を自動同期」という約束が果たされない
- **対応**: 
  - [ ] HPB API 呼び出しロジック実装
  - [ ] iCal フィード パース
  - [ ] 定期同期スケジューラ (cron job)
  - [ ] ダブルブッキング防止ロジック

#### 3. **EC・店販機能（中優先度）**
- **問題**: LP で「店販EC機能」を訴求しているが、ダッシュボードに機能がない
- **現状**: DB スキーマ存在、UI実装なし
- **影響**: デモ・営業時にEC機能を見せられない
- **対応**: 
  - [ ] `/dashboard/ec/` ディレクトリ作成
  - [ ] 商品登録UI実装
  - [ ] LINE 経由注文フロー

#### 4. **予約分析・グラフ（中優先度）**
- **問題**: LP で「売上分析」「予約分析」を訴求しているが UI なし
- **現状**: 予約データは DB に保存されるが、分析表示がない
- **影響**: ダッシュボール内で分析機能を見せられない
- **対応**:
  - [ ] `/dashboard/analytics/` 実装
  - [ ] recharts で日次予約数・売上グラフ
  - [ ] 顧客獲得・LTV 分析
