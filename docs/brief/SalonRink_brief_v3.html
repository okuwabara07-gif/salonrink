SECTION 02 · MARKET-IN

マーケットインの思考フレーム

作り手の言いたいこと（プロダクトアウト）ではなく、買い手の経営課題から逆算して設計する。

なぜ既存LPが弱いか

既存サイトの最大の問題は「誰に売っているか」が揺れていること。サロンオーナーがサイトを開いた最初の10秒で「これは自分のための道具だ」と認識できていない。ヒーローのコピーが"美容室の集客広告"に読めてしまい、本当のターゲット（サロン経営者）が離脱する。

プロダクトアウト → マーケットイン 書き換え対応表

プロダクトアウト的設計 マーケットイン的設計
ヒーローで世界観コピー＋写真ヒーローで経営課題を1秒で名指し
「コンシェルジュ体験」（手段）「再来店率 / 業務時間75%削減」（成果）
機能8つグリッド役割別ベネフィット（オーナー / スタッフ / お客様）
AI技術の自慢「メモを送るだけ、整理は不要」（体験）
プラン3階層2プランで迷わせない（フリーランス枠は残す）
連携先一覧で網羅性訴求"LINE公式アカウントだけあればOK"の潔さ
セキュリティ技術用語"LINEと同じ仕組みで守られている"

買い手が買えない3大理由（先回り解消）

  • 乗り換えが大変そう → 「既存のLINE公式アカウントに権限付与するだけ。未開設の方は開設からサポート」
  • スタッフが使いこなせない → 「LINEと同じUIの専用画面ひとつで完結」
  • お客様に違和感を与えないか → 「お客様の画面は何も変わらない」をFAQで明示
SECTION 03 · A-STRATEGY

LINEアドオン路線（A方針）の核

SalonRink Concierge は「LINE公式アカウントの上に乗る、サロン特化の業務レイヤー」として再定義されている。LINEを置き換えるツールではない。

コンセプト1行

CONCEPT
いまの LINE公式アカウントを、サロンの業務システムに。

3案から A方針を採用した経緯

Option A
LINE アドオン路線
既存LINE公式アカウントに乗っかる業務レイヤー。導入障壁が極小、LINEの安心感を借りられる。
Option B
LINE 専用ブランド路線
"LINE版"として別商品化、Web管理画面を完全廃止。差別化は出るが、既存ユーザーの混乱大。
Option C
ハイブリッド明確化
「お客様100% LINE / スタッフは専用アプリ / オーナーはWeb」と役割明示。複雑で訴求しづらい。

A方針の戦略的優位性

  • 導入障壁が極小：既存のLINE公式アカウントを使っているサロンの心理的障壁が消える
  • "新ツール導入"ではなく"アドオン"：意思決定が軽い
  • LINEの安心感を借りられる：セキュリティ訴求が簡潔になる
  • 競合戦略上も無理がない：LINEを置き換えに行かないため、LINE/ヤフー側との関係も良好
  • AOKAEの立ち位置が明確：「LINEを強化する」ポジションは独自性が高い

LINE公式アカウントの仕様前提（重要）

SPEC PREREQUISITES
・1つの LINE公式アカウントを最大 100人で運用可能
・権限は4種類（管理者・運用担当者・配信権限なし・分析閲覧権限なし）
・スタッフは LINE Official Account Manager にログインして応対（個人LINEからは直接返信不可）
・お客様から見ると、複数スタッフが返信しても 1つのサロンアカウント として届く

v2/v3 で削除した誤解表現

削除した表現問題点採用した代替
「いつものLINEから@お客様名と打つだけ」個人LINEで業務できるかのような誤解「LINEと同じUIの専用画面で、@お客様名と打つだけ」
「5分で完了」LINE未開設サロンには非現実的「お持ちなら権限付与だけ。未開設なら開設からサポート」
「LINE公式運用済み」暗黙前提ターゲット縮小「お持ちのサロンも、これからのサロンも」明示

LINE未運用サロン対応

状態比率(推定)対応
LINE公式未開設30〜40%開設代行を月額に含む（初期費用0）
持っているが未活用40〜50%リッチメニュー設計・テンプレート提供
ガッツリ活用済み10〜20%権限付与のみで連携

SECTION 15 · DO / DON'T NEW

規律 — 守ること / やらないこと

LP / LIFF / Bot メッセージ、すべての成果物に共通する規律。新担当者が最初に読むべきセクション。

コピーライティング

DO

  • 主語は必ず「サロンオーナー / スタッフ」
  • 機能名ではなく「結果として何が起きるか」を書く
  • 数値には「※モデル値・参考値」を必ず明記
  • 体験ベースの言い切り（曖昧な「〜できます」を避ける）
  • 空状態にも丁寧な案内文を書く
  • エラーは「何が起きたか + どうすればよいか」セット

DON'T

  • 来店客向けに読めるコピー（「サロンに、専属コンシェルジュを」型）
  • AI 技術用語（「自動構造化」「ベクトル検索」など）
  • セキュリティ技術用語を前面に出す（AES-256, TLS 1.3, RLS）
  • 「いつものLINEで」など個人 LINE で業務できる誤解表現
  • 「5分で完了」など LINE 未開設サロンに非現実的な表現
  • 「SalonRink (LINE Edition)」表記

デザイン

DO

  • 余白を恐れず使う（密度より静寂）
  • 数値は JetBrains Mono で揃える（¥4,000 / 60分 / 14日）
  • 見出しは Noto Serif JP で品格を出す
  • アクセント色は 1画面に 2-3 要素まで
  • 美容師向けは LINEグリーン、顧客向けはコーラルで分離
  • 主要 CTA は画面下部、タップ領域 44pt 以上

DON'T

  • 過剰な絵文字、装飾アイコンの羅列
  • 不要なグラデーション
  • 派手なアニメーション、紙吹雪、confetti
  • 「コンシェルジュくん」型の AI キャラ擬人化
  • 機能を平等に並列表示（優先度のない UI）
  • ダークモードのデフォルト化（warm light モードのみ）

実装

DO

  • commit message は英語、簡潔に（refactor/feat/fix プレフィックス）
  • Vercel チーム デフォルトビルドマシン = Standard 維持
  • キレイ鶴見店で実機動作確認してから他サロン営業
  • LP コピーは実装可能な範囲内に留める
  • 機能を訴求する前に「結果」を訴求する
  • API 鍵・トークンは Vercel 環境変数のみ、コミットしない

DON'T

  • commit message に全角括弧（）使用（dquote> シェルハング原因）
  • Hobby プランで商用機能を運用（AdSense / 決済 / アフィリ）
  • 個人情報の Supabase テーブル直接編集
  • 本番 LINE OA に未検証 Bot を接続
  • LIFF 内で localStorage / sessionStorage に依存
  • Claude API を Haiku 以外で大量呼び出し（コスト爆発）

AI 技術アピール（"自動構造化" "自動解析"）AI は目的ではなく手段
