## セクション 2: 導入前後の成果（Before/After）

**デザイン方針**  
3カラム構成で視覚的に数値改善を表現。各カードはシンプルで読みやすく。

**レイアウト**  
3つのカード（横並び）：

### カード 1: 予約確定率

```
Before: 68%
説明：ホットペッパー・LINE・電話が混在、
      重複予約が月 12 件発生

↓ SALOMÉ 導入

After: 92%
説明：システムが自動で重複をブロック

成果：「月 12 件の予約取りこぼしを防止」
```

### カード 2: 顧客再来率

```
Before: 45%
説明：LINEリマインドを手動で送信、
      フォローが属人的

↓ SALOMÉ 導入

After: 68%
説明：来店後 24h に自動でフォロー送信

成果：「新規顧客の定着率が 1.5 倍に」
```

### カード 3: 月額売上

```
Before: ¥680,000
説明：施術売上のみ

↓ SALOMÉ 導入

After: ¥850,000
説明：施術売上 + EC販売 ¥170k/月

成果：「追加売上 ¥170k/月を実現」
```

## セクション 3: 機能紹介

**デザイン方針**  
3カラム構成。各機能にアイコン + タイトル + 説明を配置。

### 機能 1: LINE予約（ワンタップで予約完結）

**アイコン：** 📱

**説明**  
顧客がLINEリッチメニューから予約。スタッフは自動で Supabase に同期。ホットペッパーとの重複をシステムが自動ブロック。

### 機能 2: 顧客カルテ（来店履歴を一元管理）

**アイコン：** 📋

**説明**  
施術メニュー・スタイル写真・次回希望日を記録。AIが顧客の好みを学習し、推奨メニューを提案。ポイント自動加算で顧客ロイヤリティ向上。

### 機能 3: ECストア（店販・アフィリで追加収入）

**アイコン：** 🛍

**説明**  
自社商品とアフィリ商品を同一カタログで販売。配送代行も対応。顧客の来店後 LINE で自動推奨メール送信。

## 技術仕様

**ファイル**  
- Create: `app/salons/tsurumi/page.tsx`
- Update: `app/layout.tsx`（ナビゲーション リンク追加、必要に応じて）

**コンポーネント戦略**  
- Server Component（`page.tsx`）でページを構成
- 各セクションは inline styles で統一スタイル
- 既存 SALOMÉ LP のカラースキーム・フォント・間隔を踏襲

**データ**  
- 実績数字はシミュレーション値
- ハードコード（動的データベース連携なし）

**ナビゲーション**  
- `/salons/tsurumi` から `/register` へのリンク
- `/page.tsx` ヘッダーに「キレイ鶴見店」リンク追加（検討）
