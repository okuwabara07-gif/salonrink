# 引き継ぎ資料：予約管理画面リデザイン

## 概要

サロン向け予約管理画面（旧：縦リスト型）を、**横タイムライン × スタッフ行**のカレンダー型レイアウトへリデザインしたものです。1日の予約状況を一目で把握し、空き枠の確認・予約の状態変更・新規予約追加までを1画面で完結させることが目的です。

参照元：`salonrink.com/dashboard/booking`（リスト型）
参考にしたUXパターン：サロン業界で標準的な横タイムライン型予約管理（ただし特定ブランドのUIは複製せず、本プロジェクト独自の世界観で再設計）

---

## このバンドルについて

この資料に同梱されているHTML/CSS/JSファイルは、**意図する見た目と挙動を示すデザインリファレンス（HTMLで作られたプロトタイプ）**です。プロダクションコードとしてそのままコピー&ペーストするためのものではありません。

タスクは、これらのHTMLデザインを、**ターゲットコードベースの既存環境（React / Vue / Next.js など）**へ再実装することです。既存の状態管理・ルーティング・デザインシステムの慣習に沿って実装してください。コードベースが未確立の場合は、プロジェクトに最適なフレームワーク（Next.js + React 推奨）を選定して実装してください。

---

## フィデリティ

**High-fidelity (hifi)** — 色・タイポグラフィ・余白・インタラクションまで含めた本番想定のモック。コードベースの既存ライブラリ・パターンを用いつつ、可能な限りピクセルパーフェクトに再現してください。

## 画面構成

### 1. トップバー (`.topbar`)
- 左：ブランドロゴ（"SALON RINK"、明朝、letter-spacing 0.08em）＋ パンくず（"管理 / 予約"）
- 中央：店舗名（"鶴見店 ・ オシャレな白髪染め専門店"、小さい字、letter-spacing 0.12em）
- 右：ヘルプボタン ＋ ユーザーチップ（アバター＋ "店長 / 菊地"）
- 下端 1px ボーダー `--line-soft`

### 2. コントロールバー (`.controlbar`)
横一列。flex wrap。
- ページタイトル "予約管理"（明朝、28px）＋ 上に小さなオーバーライン "RESERVATIONS"
- 日付ナビ：◀ / 日付（明朝、"5月14日 (木)") / ▶ / "今日" ピル
- セグメントコントロール：日 / 週 / 月（明朝、アクティブは黒地クリーム文字）
- 右側フィルタ群（filterset、margin-left:auto）：
  - 検索ボックス（虫眼鏡アイコン）"お客様名 / メニュー"
  - 担当セレクト：全員 / スタッフ各名
  - 経路セレクト：全て / ウェブ予約 / 電話 / 直接来店 / リピーター
  - 状態セレクト：全て / 確定 / 仮予約 / 来店中 / 完了 / 見送り
  - 印刷ボタン
  - 新規予約ボタン（プライマリ、黒地）

### 3. サマリーストリップ
左から、縦区切り線で区切られた KPI セル：
- 本日の予約数
- 確定（件数）
- 来店中（件数、青）
- 新規顧客（名）
- 見込み売上（¥ 円、モノスペース）
- 右端：現在時刻（赤、モノスペース）

### 4. タイムライン本体 (`.timeline-card`)
クリーム白カード、角丸18px、淡いボーダー、淡い影。
内側に CSS Grid：左 196px のスタッフ列、右側は時間軸（10:00〜20:00、1時間 = 160px）。

**ヘッダー行**（時間軸）
- 左セル："時間" ラベル
- 右セル：1時間ごとの目盛り（モノスペース 12px）、右下に "予約 N 件" の小さなカウンタ

**サマリー行**（30分刻みの受付状況）
- 左セル："受付状況 / 30分単位 / 全Nブースト"
- 右セル：30分ごとのセル。上段に "残N"、下段に大きく予約数（モノスペース 16px）。満枠なら赤、空きならグレー。

**スタッフ行**（各スタッフ1行、最低高さ76px）
- 左セル：色付きの円アバター（イニシャル）、名前（明朝）、ロール（"店長 / カラリスト" 等）、右端に統計（X件 / Y.Yh）
- 右セル：絶対配置で予約カード。背景に1時間/30分のグリッド線。空行のときは "ドラッグ＆ドロップで予約を割り当て" のプレースホルダ。

### 5. 予約カード (`.bk`)
タイムライン上の絶対配置要素（top:6px / bottom:6px、left/width はパーセント）。
- 角丸6px、白背景、淡いボーダー、**左3px のステータス色ライン**
- ホバーで1pxリフトと影
- 選択時は2px黒アウトライン
- 中身：
  - 1行目：時間（モノスペース）＋ ステータスピル（背景色付き）
  - 2行目：お客様名（明朝、末尾に小さく "様"）
  - 3行目：メニュー名（グレー、1行省略）
  - 4行目：経路バッジ（"WEB" "TEL" "店頭" "RPT" のモノスペース短縮形）＋ タグ（VIP / 新規 / 指名 / 要割当）
- 60分未満のカードは `is-short` モディファイア：メニューとフッターを非表示にして名前と時間のみ。

### 6. 現在時刻ライン (`.tl-now`)
タイムライン上を縦に貫く赤い1pxライン。上部に丸ぽち、ラベル "13:42"（モノスペース、赤枠付き白地）。

### 7. 詳細パネル (`.detail`、右側 380px、選択時のみ表示)
カード選択でメインを `grid-template-columns: 1fr 380px` に切り替えてスライドイン。
- ヘッダー："予約詳細" ＋ 閉じるボタン
- 顧客カード：丸アバター（明朝）／ 顧客名 ＋ タグ ＋ 電話番号
- フィールド行（86px ラベル × 1fr バリュー、点線で区切り）：
  - ステータス（ピル ＋ 他ステータスへの遷移ボタン群）
  - 日時（"14:30 – 16:30 (120分)"）
  - 担当（色アバター ＋ 氏名 ＋ ロール）
  - メニュー
  - 料金（モノスペース、¥18,700）
  - 予約経路
  - メモ（薄いカードに囲んで表示）
- フッターアクション：編集 / 来店受付（プライマリ）

### 8. 新規予約モーダル (`.modal`)
中央モーダル、520px幅、角丸18px。
- フィールド：お客様名・電話番号 ／ 担当スタッフ・予約経路 ／ 開始時刻（時・分）・所要時間 ／ メニュー ／ メモ
- フッター："キャンセル" / "仮予約として追加"（プライマリ）

## インタラクション

| 操作 | 挙動 |
|---|---|
| 予約カードをクリック | 詳細パネルを開く / 同じものをもう一度押すと閉じる |
| 詳細パネルのステータス変更ボタン | 状態を即時変更（楽観的更新） |

| "今日" ボタン | 今日へジャンプ |
| 日 / 週 / 月 | ビュー切替（現状は日ビューのみ実装） |
| 検索入力 | 顧客名・メニューで部分一致フィルタ |
| 担当 / 経路 / 状態セレクト | フィルタ |
| 新規予約ボタン | モーダル表示 → 入力 → 仮予約として追加 |

## 実装上の注意

1. **タイムラインの絶対配置**：予約カードは `position: absolute` で `left`/`width` をパーセンテージ指定（時刻軸の総分数で割った値）。レスポンシブで時刻軸の幅が変わっても破綻しないようにこの方式を採用。

2. **背景グリッド**：行の背景は `repeating-linear-gradient` 2 枚重ねで1時間ごとの実線と30分ごとの淡線を表現。スクロールしてもズレないように `background-size` を `--hour-w` 基準で指定。

3. **横スクロール**：時間軸は左の固定スタッフ列のとなりで横スクロール可能。`overflow-x: auto` を内側ラッパに付与。

4. **印刷**：印刷時はサマリー帯と詳細パネル・Tweaks パネルを `@media print` で非表示にすると見やすくなります（現状未指定 → 追加してOK）。

5. **時間の単位**：すべて「00:00からの経過分」で扱っています。日跨ぎや営業時間設定（開店9:00 / 閉店21:00 等）を可変にする場合は `TIME_START` / `TIME_END` を設定値として吸い出してください。

6. **アクセシビリティ**：セグメントコントロールに `role="tablist"`、閉じるボタンに `aria-label` を付けています。詳細パネルにはフォーカストラップとEscキー閉じるを追加実装してください。

7. **ブランディング配慮**：参考画像にあった他社サービス（SALON BOARD / Hot Pepper Beauty 等）のロゴ・色・アイコンは意図的に再現していません。ソース表示は "WEB" "TEL" "店頭" "RPT" のような汎用的な短縮形にしています。本プロジェクト独自の連携サービス名が確定したら、`data.js` の `SOURCE_LABEL` を差し替えてください。
